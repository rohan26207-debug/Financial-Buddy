# Finance Buddy — Android Build Guide

This folder contains a native Android (Java) WebView wrapper that bundles the React app as offline assets, so you can produce a self-contained APK with **no internet permission** required.

## Project structure

```
android/
├── app/
│   ├── build.gradle
│   ├── proguard-rules.pro
│   └── src/main/
│       ├── AndroidManifest.xml
│       ├── assets/                 # React build output is copied here
│       ├── java/com/financebuddy/app/MainActivity.java
│       └── res/
│           ├── drawable/ic_launcher_foreground.xml
│           ├── layout/activity_main.xml
│           ├── mipmap-anydpi-v26/ic_launcher{,_round}.xml
│           ├── values/{colors,strings,themes,ic_launcher_background}.xml
│           └── xml/{network_security_config,backup_rules,data_extraction_rules}.xml
├── build.gradle
├── settings.gradle
└── gradle.properties
```

## Build steps

### 1. Build the React app and sync into Android assets

From the repo root:

```bash
bash scripts/build-android-assets.sh
```

This will:
1. Run `PUBLIC_URL=. yarn build` inside `frontend/` to produce a self-contained build with relative asset paths.
2. Wipe `android/app/src/main/assets/` and copy the new build into it.

After this, `android/app/src/main/assets/index.html` should exist along with `static/css/...` and `static/js/...` folders.

### 2. Open the project in Android Studio

1. Launch Android Studio (Iguana 2023.2.1 or newer recommended — Gradle 8.2, AGP 8.2).
2. **File → Open…** and pick the `/android` folder.
3. Let it download Gradle. The first sync will also generate the Gradle wrapper (`gradlew`) automatically.
4. If prompted, install **Android SDK Platform 34**, **Build Tools 34.0.0**, and **Platform-Tools**.

### 3. Run on a device or emulator

- Connect an Android phone with USB debugging enabled (or start an emulator with API 26+).
- Click **Run ▶** in Android Studio. The app installs and opens.
- The app loads `file:///android_asset/index.html` inside a WebView. All data lives in WebView `localStorage`, so it works fully offline (turn on Airplane Mode to verify).

### 4. Build a release APK

In Android Studio:

- **Build → Generate Signed Bundle / APK… → APK → Next**.
- Create a keystore the first time (keep it safe — needed for future updates).
- Choose `release` build variant.
- The signed APK will be in:
  `android/app/build/outputs/apk/release/app-release.apk`

For an unsigned debug APK, just run **Build → Build APK(s)**; the file is at:
`android/app/build/outputs/apk/debug/app-debug.apk`.

Or from the command line (after Gradle wrapper is generated by Android Studio):

```bash
cd android
./gradlew assembleDebug      # debug APK
./gradlew assembleRelease    # release APK (unsigned unless signing config is added)
```

## Configuration notes

| Setting          | Value                                                |
| ---------------- | ---------------------------------------------------- |
| Application ID   | `com.financebuddy.app`                               |
| Min SDK          | 26 (Android 8.0)                                     |
| Target SDK       | 34                                                   |
| Compile SDK      | 34                                                   |
| Java             | 17                                                   |
| Internet         | **Not requested** — app is fully offline             |
| Data persistence | WebView `localStorage` (key `finance_buddy_data_v1`) |

The WebView is configured with:

- `setJavaScriptEnabled(true)`
- `setDomStorageEnabled(true)` — enables `localStorage`
- `setDatabaseEnabled(true)`
- `setAllowFileAccess(true)`
- Mixed-content blocking on (no HTTP allowed)
- Hardware back button maps to in-app history

## Updating the bundled web app

Whenever the React code changes, just rerun:

```bash
bash scripts/build-android-assets.sh
```

…and rebuild the APK in Android Studio (or via `./gradlew assembleDebug`).

## Troubleshooting

- **Blank screen on launch:** make sure `android/app/src/main/assets/index.html` exists. Re-run the build script.
- **Buttons appear but routes break:** the React app uses `HashRouter`, so URLs look like `index.html#/loans`. This is intentional and required for `file://` loading.
- **`gradlew` not found:** open the project once in Android Studio and let it sync — it will create `gradlew`/`gradlew.bat` and the wrapper jar automatically.
- **Build fails because of SDK:** open SDK Manager in Android Studio and install API 34.
